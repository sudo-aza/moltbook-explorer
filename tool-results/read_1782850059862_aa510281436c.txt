    25→
    26→Six phases. Rules and guardrails are embedded in each phase where they apply.

    28→### Phase 1 - BRIEF
    29→
    30→Confirm with the user:
    31→- Document type (journal article, thesis chapter, conference paper, technical report)
    32→- Template requirement (IEEE, ACM, custom class, or plain `article`)
    33→- Bibliography format (natbib superscript, numeric brackets, biblatex)
    34→- Expected length → if >5 pages or 3+ complex elements (wide tables, block equations, TikZ), split into `\input{}` modules (~500 lines each)

    36→**Recognising the document's personality:**
    37→
    38→| Personality | Examples | Key concern |
    39→|-------------|----------|-------------|
    40→| Scholarly | Journal articles, conference proceedings | Academic conventions, bibliography accuracy |
    41→| Utilitarian | Technical reports, manuals, specs | Information density + scannability |

    43→| Expressive | Portfolios, brand guidebooks | Bold typographic choices |

    45→### Phase 2 - SETUP
    46→
    47→**Title Page (Cover) Rules:**
    48→- **Academic route covers are generated via HTML/Playwright**, using Templates 03-04 and 06 from `typesetting/cover.md`. Templates 03-04 replicate LaTeX title page aesthetics (dark backgrounds, serif titles, symmetric layouts) in HTML/CSS. **Template 06 (Institutional)** is for thesis proposals, dissertations, and formal institutional submissions (white bg + black border frame).
    49→- **Pipeline:** Generate body PDF via Tectonic (no title page in `.tex`) → Generate cover HTML (Template 03/04/05/06) → Playwright `page.pdf()` → Merge cover as page 0 via pypdf
    50→- **Template selection:** For thesis proposals (开题报告), dissertations (毕业论文), and institutional submissions → **default to Template 06**. For research papers, preprints, journal submissions → Templates 03-04.
    51→- **NEVER use `\maketitle`** - it produces ugly default output with cramped spacing
    52→- **NEVER use `\begin{titlepage}...\end{titlepage}`** - the cover is generated separately via HTML/Playwright
    53→- **NEVER use LaTeX TikZ overlay for full-page covers** - TikZ `current page` coordinates are unreliable with `margin=0pt`, causing backgrounds to not fill the page (right/bottom white edges). HTML/CSS full-bleed is pixel-exact.
    54→- Title page is OPTIONAL - skip it for short documents (≤ 2 pages), letters, memos, or when content scanning is priority
    55→- **`\tableofcontents` must be the FIRST page** of the body PDF (after merge, it becomes page 2). If no TOC, content starts on page 1 of the body PDF
    56→
    57→
    58→**Cover Generation Pipeline (Academic route):**
    59→```
    60→1. Write .tex WITHOUT any title page
    61→2. Run poster_validate.py check-tex on .tex file - fix table overflow / image width ERRORs
    62→3. Compile with tectonic → body.pdf
    63→4. Write cover HTML using Template 03/04/05/06 from typesetting/cover.md (Template 06 for thesis proposals/dissertations)
    64→5. Run poster_validate.py check-html on cover HTML - fix any ERRORs
    65→6. Run cover_validate.js on cover HTML - fix any text-line overlaps
    66→7. Render cover HTML → PDF via Playwright (`html2poster.js`) — **NOT `html2pdf-next.js`** (which converts absolute→static and destroys cover layout)
    67→8. Merge: insert cover as page 0 of body PDF via pypdf
    68→```
    69→
    70→**Cover HTML → PDF rendering:**
    71→```bash
    72→# ALWAYS use html2poster.js for cover rendering (NOT html2pdf-next.js)
    73→# Cover pages use position:absolute layout — html2pdf-next.js pre-render hooks
    74→# convert absolute→static and destroy the layout. html2poster.js preserves it.
    75→node "$PDF_SKILL_DIR/scripts/html2poster.js" cover.html --output cover.pdf --width 794px
    76→```
    77→
    78→Or from Python:
    79→```python
    80→import subprocess, os
    81→
    82→def render_cover(html_path, pdf_path):
    83→    """
    84→    Render HTML cover to PDF via html2poster.js.
    85→    
    86→    ⚠️ ALWAYS use html2poster.js for covers (NOT html2pdf-next.js).
    87→    Cover HTML uses position:absolute for layout. html2pdf-next.js pre-render
    88→    hooks convert absolute→static to prevent multi-page overlap, which
    89→    destroys cover layouts. html2poster.js preserves absolute positioning.
    90→    """
    91→    scripts_dir = os.path.join(PDF_SKILL_DIR, 'scripts')  # PDF_SKILL_DIR from SKILL.md § Script Path Setup
    92→    subprocess.run([
    93→        'node', os.path.join(scripts_dir, 'html2poster.js'),
    94→        html_path, '--output', pdf_path,
    95→        '--width', '794px',
    96→    ], check=True)
    97→```

    99→**Merge cover + body:**
   100→```python
   101→from pypdf import PdfReader, PdfWriter
   102→
   103→A4_W, A4_H = 595.28, 841.89  # A4 in points
   104→
   105→def normalize_page_to_a4(page):
   106→    """Scale a page to A4 if its dimensions don't match."""
   107→    box = page.mediabox
   108→    w, h = float(box.width), float(box.height)
   109→    if abs(w - A4_W) > 2 or abs(h - A4_H) > 2:
   110→        page.scale_to(A4_W, A4_H)
   111→    return page
   112→
   113→writer = PdfWriter()
   114→cover_page = normalize_page_to_a4(PdfReader('cover.pdf').pages[0])
   115→writer.add_page(cover_page)
   116→for page in PdfReader('body.pdf').pages:
   117→    writer.add_page(page)
   118→with open('final.pdf', 'wb') as f:
   119→    writer.write(f)
   120→```

   124→> **⚠️ Why HTML/Playwright covers?** LaTeX TikZ `remember picture, overlay` with `margin=0pt` frequently fails to fill the page (right/bottom edges show white). HTML/CSS with `@page { margin: 0 }` and full-bleed background is pixel-exact, with zero ambiguity. This also unifies all three routes (Report, Creative, Academic) under one cover system.

   161→**Guardrails for SETUP:**
   162→- `hyperref` must load after virtually every other package - option clashes are the #1 preamble bug
   163→- When using a Scenario template (A/B/C) or Resume template, use that template's own preamble instead
   164→- `babel` and `polyglossia` are incompatible - load only one
   165→- CJK: `\usepackage{ctex}` - Tectonic auto-downloads fonts, zero manual setup
   166→- System fonts via `\setmainfont{}`: probe first with `fc-list :lang=XX`
   167→- ** Margin symmetry:** `\geometry{left=X, right=X}` - left and right MUST be equal. Asymmetric margins = off-center content = critical bug
   168→- ** Minimum margins with fancyhdr:** When using `fancyhdr` for headers/footers, `geometry` margins must leave enough room. **Minimum: `top >= 2.0cm`, `bottom >= 1.8cm`**. Also set `\setlength{\headheight}{14pt}` in the preamble. Margins smaller than this cause headers/footers to be pushed outside the page boundary (negative y-coordinates), making them invisible in print.
   169→- ** Quotation marks (English):** NEVER use straight quotes `"..."`。English text must use LaTeX curly quotes: ` ``left quote'' ` for double, `` `single' `` for single. Straight `"` in LaTeX means right double quote only.
   170→- ** Quotation marks (Chinese — CRITICAL):** Chinese quoted text like "北漂" MUST use Unicode smart quotes "…" (U+201C/U+201D) directly in the `.tex` source. **NEVER use ASCII `"` for Chinese quotes** — LaTeX interprets `"` as a right double quote (`"`), so `"北漂"` renders as `"北漂"` (two right quotes, no left quote). The correct LaTeX source is: `"北漂"` (literal Unicode characters). `\usepackage{csquotes}` is a safety net but does NOT fix raw ASCII `"` in Chinese text.
   171→  - **Scope:** This rule applies ONLY to Chinese-language body text. Do NOT replace `"` in English paragraphs (use ` ``...'' ` instead), `verbatim`/`lstlisting`/`minted` environments, `\texttt{}`/`\verb||`/`\url{}`/`\href{}{}` arguments, or BibTeX `.bib` field values.
   172→- ** Title page isolation:** Cover is generated via HTML/Playwright and merged as page 0 via pypdf - isolation is inherent in the merge pipeline. `\tableofcontents` should be the first page of the `.tex` body. Verify: does TOC start on the page immediately after the cover in the merged PDF?
   173→- ** TOC requires a cover page:** Unless the user explicitly requests no cover, if the document has `\tableofcontents`, it MUST have a cover page. Structure: Cover (page 1) → TOC (page 2) → Content (page 3+). Do not generate a TOC without a preceding cover page. This rule is consistent with `briefs/report.md`.

   186→### Phase 3 - BUILD
   187→
   188→Write LaTeX content: sections, equations, figures, tables, bibliography.
   189→
   190→**→ MUST READ: `typesetting/overflow.md`** for LaTeX-specific overflow patterns (tabularx, adjustbox, widowpenalty, etc.). Key rules:
   191→- Tables: always use `tabularx` or `tabular*` with `\columnwidth` constraint - never plain `tabular` for 5+ columns
   192→- Images: always `\includegraphics[max width=\columnwidth, max height=0.4\textheight]` or `adjustbox` - the `max height` prevents a single figure from occupying an entire page
   193→- Orphans/widows: set `\widowpenalty=10000` and `\clubpenalty=10000`
   194→- Long tables: use `longtable` with `\endhead` for header repetition on every page
   195→
   196→**Embedding components**: If the document needs heavy math, diagrams, or algorithms, refer to the Scenario sections below and embed their templates/environments into your `.tex` file. Scenarios are not separate tasks - they are building blocks for Phase 3.
   197→
   198→**Content density guidance (for textbooks, lecture notes, tutorials):**
   199→
   200→Documents meant for learning should maintain a healthy balance of prose and formal elements. A page full of equations with no explanation reads like a reference manual, not a textbook.
   201→
   202→Guidelines:
   203→- Every equation/equation group should be **preceded** by a sentence explaining what it represents and **followed** by a sentence interpreting the result or stating its significance
   204→- Every figure should have (a) a descriptive `\caption{}` and (b) at least one sentence in the surrounding text referencing it
   205→- Every theorem/definition should be followed by an intuitive explanation or worked example before proceeding to the next theorem
   206→- Avoid stacking 3+ formal elements (equations, figures, tables) with zero narrative text between them
   207→- For worked examples: state the problem, show the solution steps, then summarize the key takeaway
   208→
   209→**When high density is acceptable**: research papers (especially methods sections where equation groups naturally cluster), formula sheets, reference appendices, and conference papers with tight page limits. In these cases, prioritise completeness over readability padding.
   210→
   211→**Source hygiene - catch these model-generation slips:**
   212→- **Prohibited**: emoji glyphs (tofu), markdown `*asterisk*` formatting (compile errors)
   213→- **Use instead**: `\textbf{bold}`, `\emph{emphasis}`
   214→
   215→**Table placement - prevent header orphans:**
   216→- Short tables (≤15 rows): wrap in `\begin{table}[htbp]` - LaTeX keeps it together
   217→- Long tables (>15 rows): use `longtable` with repeated header:
   218→```latex
   219→\usepackage{longtable}
   220→\begin{longtable}{lll}
   221→\toprule
   222→Header 1 & Header 2 & Header 3 \\
   223→\midrule
   224→\endfirsthead
   225→\toprule
   226→Header 1 & Header 2 & Header 3 \\  % repeated on continuation pages
   227→\midrule
   228→\endhead
   229→\bottomrule
   230→\endfoot
   231→Row 1 & data & data \\
   232→Row 2 & data & data \\
   233→\end{longtable}
   234→```
   235→- **Never** let a table header sit alone at the bottom of a page with no data rows following it
   236→
   237→**Table width management - prevent column overflow (⚠️ CRITICAL):**
   238→
   239→Tables overflowing the column width is the most common LaTeX layout bug in dual-column papers. The table looks fine in single-column preview but clips in IEEE/ACM two-column format.
   240→
   241→**Prevention strategy (in priority order):**
   242→
   243→1. **Use `tabular*` or `tabularx` to constrain width** (RECOMMENDED):
   244→```latex
   245→% tabular* - fixed total width, stretches inter-column space
   246→\begin{table}[htbp]
   247→\centering
   248→\caption{Results.}\label{tab:results}
   249→\begin{tabular*}{\columnwidth}{@{\extracolsep{\fill}} l cccc}
   250→\toprule
   251→Method & P@10 & R@10 & NDCG@10 & MRR \\
   252→\midrule
   253→Ours   & \textbf{0.082} & \textbf{0.054} & \textbf{0.043} & \textbf{0.029} \\
   254→\bottomrule
   255→\end{tabular*}
   256→\end{table}
   257→
   258→% tabularx - fixed total width, X columns auto-stretch
   259→\usepackage{tabularx}
   260→\begin{tabularx}{\columnwidth}{l X X X X}
   261→...
   262→\end{tabularx}
   263→```
   264→
   265→2. **Reduce font size inside table** (common in conference papers):
   266→```latex
   267→\begin{table}[htbp]
   268→\centering
   269→\small            % or \footnotesize for very wide tables
   270→\caption{Comparison.}\label{tab:comp}
   271→\begin{tabular}{lcccccccc}
   272→...
   273→\end{tabular}
   274→\end{table}
   275→```
   276→
   277→3. **`\resizebox` as last resort** (scales the entire table to fit):
   278→```latex
   279→\begin{table}[htbp]
   280→\centering
   281→\caption{Full results.}\label{tab:full}
   282→\resizebox{\columnwidth}{!}{%
   283→\begin{tabular}{lcccccccc}
   284→\toprule
   285→...
   286→\bottomrule
   287→\end{tabular}
   288→}
   289→\end{table}
   290→```
   291→⚠️ `\resizebox` scales fonts too - verify the smallest text is still readable (≥ 6pt effective).
   292→
   293→4. **Span both columns** for genuinely wide tables (8+ data columns):
   294→```latex
   295→\begin{table*}[t]   % table* spans full width in twocolumn
   296→\centering
   297→\caption{Cross-dataset results.}\label{tab:cross}
   298→\begin{tabular}{lccccccccccc}
   299→...
   300→\end{tabular}
   301→\end{table*}
   302→```
   303→
   304→**Decision checklist before writing any table:**
   305→| Data columns | Single-column (`\columnwidth`) | Action |
   306→|-------------|-------------------------------|--------|
   307→| ≤ 4 | Fits comfortably | Normal `tabular` |
   308→| 5-6 | Tight fit | `\small` + `tabular*{\columnwidth}` |
   309→| 7-8 | Won't fit | `\footnotesize` + `tabular*`, or `\resizebox` |
   310→| ≥ 9 | Definitely won't fit | Use `table*` (full width), or split into two tables |
   311→
   312→**Never**: use a plain `tabular` with 8+ columns in a two-column paper without width constraint - it WILL overflow.
   313→
   314→**⚠️ CRITICAL: `\resizebox{\columnwidth}` NOT `\resizebox{\textwidth}` in two-column layouts!**
   315→In dual-column documents (`twocolumn`, `sigconf`, etc.), `\textwidth` = **full page width** (both columns), while `\columnwidth` = **single column width**. Using `\resizebox{\textwidth}` inside a `table` (single-column float) scales the table to the full page width, causing it to overflow the column boundary by ~50%. **Always use `\resizebox{\columnwidth}` for single-column floats.** Only use `\resizebox{\textwidth}` inside `table*` (full-width float).
   316→
   317→```latex
   318→% ❌ WRONG in two-column layout
   319→\begin{table}[t]
   320→  \resizebox{\textwidth}{!}{% <-- \textwidth = full page, table overflows column!
   321→    \begin{tabular}{lcccccccc} ... \end{tabular}}
   322→\end{table}
   323→
   324→% ✅ CORRECT for single-column float
   325→\begin{table}[t]
   326→  \resizebox{\columnwidth}{!}{% <-- \columnwidth = single column width
   327→    \begin{tabular}{lcccccccc} ... \end{tabular}}
   328→\end{table}
   329→
   330→% ✅ CORRECT for full-width float
   331→\begin{table*}[t]
   332→  \resizebox{\textwidth}{!}{% <-- \textwidth OK here because table* spans both columns
   333→    \begin{tabular}{lcccccccc} ... \end{tabular}}
   334→\end{table*}
   335→```

   339→**Equation overflow prevention (⚠️ CRITICAL for dual-column papers):**
   340→
   341→Long equations are the **#2 overflow source** after tables in dual-column papers. Column width in ACM `sigconf` is ~241pt; in IEEE `twocolumn` ~252pt. Many standard math expressions exceed this.
   342→
   343→**Overflow patterns and fixes:**
   344→
   345→| Pattern | Problem | Fix |
   346→|---------|---------|-----|
   347→| Two equations side-by-side with `\quad` | Combined width > column | Split into `align` with one equation per line |
   348→| Deep fraction nesting (softmax, attention) | Denominator sum too wide | Use `\smash` + separate definition, or `split` |
   349→| Long subscripts/superscripts with `\text{}` | `\text{collab}`, `\text{social}` are wide | Use short math abbreviations: `c`, `s`, or define `\newcommand` |
   350→| `equation` with multiple terms separated by `\quad` | Horizontal overflow | Use `aligned` inside `equation`, or `align` |
   351→
   352→**Rule M1 — Never put two independent equations on one line in dual-column:**
   353→```latex
   354→% ❌ WRONG — two full equations on one line, guaranteed overflow in sigconf
   355→\begin{equation}
   356→\mathbf{e}_u^{(l+1)} = \sum_{i} \frac{1}{\sqrt{|N_R(u)|\cdot|N_R(i)|}} \mathbf{e}_i^{(l)}, \quad
   357→\mathbf{e}_i^{(l+1)} = \sum_{u} \frac{1}{\sqrt{|N_R(i)|\cdot|N_R(u)|}} \mathbf{e}_u^{(l)}
   358→\end{equation}
   359→
   360→% ✅ CORRECT — split into aligned or separate equations
   361→\begin{align}
   362→\mathbf{e}_u^{(l+1)} &= \sum_{i \in \mathcal{N}_R(u)} \frac{\mathbf{e}_i^{(l)}}{\sqrt{|\mathcal{N}_R(u)| \cdot |\mathcal{N}_R(i)|}}, \label{eq:collab_u} \\
   363→\mathbf{e}_i^{(l+1)} &= \sum_{u \in \mathcal{N}_R(i)} \frac{\mathbf{e}_u^{(l)}}{\sqrt{|\mathcal{N}_R(i)| \cdot |\mathcal{N}_R(u)|}}. \label{eq:collab_i}
   364→\end{align}
   365→```
   366→
   367→**Rule M2 — Wide fractions: use `split` or `multline`:**
   368→```latex
   369→% ❌ WRONG — softmax with long denominator
   370→\begin{equation}
   371→\alpha_{uv} = \frac{\exp(\text{LeakyReLU}(\mathbf{a}^{\top}[\mathbf{W}\mathbf{e}_u \| \mathbf{W}\mathbf{e}_v]))}{\sum_{k \in \mathcal{N}_S(u)} \exp(\text{LeakyReLU}(\mathbf{a}^{\top}[\mathbf{W}\mathbf{e}_u \| \mathbf{W}\mathbf{e}_k]))}
   372→\end{equation}
   373→
   374→% ✅ CORRECT — define numerator/denominator separately
   375→\begin{equation}
   376→\alpha_{uv} = \frac{\exp\bigl(f(\mathbf{e}_u, \mathbf{e}_v)\bigr)}{\sum_{k \in \mathcal{N}_S(u)} \exp\bigl(f(\mathbf{e}_u, \mathbf{e}_k)\bigr)},
   377→\end{equation}
   378→\text{where } f(\mathbf{e}_u, \mathbf{e}_v) = \text{LeakyReLU}\bigl(\mathbf{a}^{\top} [\mathbf{W}\mathbf{e}_u \| \mathbf{W}\mathbf{e}_v]\bigr).
   379→```
   380→
   381→**Rule M3 — Self-check: if `equation` body > 60 characters (excluding `\label`), it probably overflows dual-column:**
   382→This is a quick mental check. If the raw LaTeX math string is very long, it almost certainly won't fit in ~241pt. Use `align`, `split`, `multline`, or factor out sub-expressions.
   383→
   384→**Rule M4 — Contrastive loss / InfoNCE: always use `multline` or `split`:**
   385→Contrastive losses with `\frac{\exp(...)}{\sum \exp(...)}` inside `\log` are notoriously wide. Always break them across lines:
   386→```latex
   387→\begin{multline}
   388→\mathcal{L}_{\text{SSL}}^u = -\log \frac{\exp\bigl(\text{sim}(\mathbf{z}_u', \mathbf{z}_u'') / \tau\bigr)}
   389→{\sum_{v \neq u} \exp\bigl(\text{sim}(\mathbf{z}_u', \mathbf{z}_v'') / \tau\bigr)}.
   390→\end{multline}
   391→```

   395→**Algorithm overflow prevention (dual-column papers):**
   396→
   397→Algorithm boxes with long `\KwInput` lines or verbose pseudocode frequently overflow column width.
   398→
   399→**Rule A1 — Always set `\SetAlFnt{\small}` and limit line width:**
   400→```latex
   401→\SetAlFnt{\small}           % Smaller font inside algorithm
   402→\SetAlCapFnt{\small}        % Smaller caption font
   403→\SetAlCapNameFnt{\small}    % Smaller "Algorithm N:" prefix
   404→```
   405→
   406→**Rule A2 — Break long Input/Output lines:**
   407→```latex
   408→% ❌ WRONG — all parameters on one line
   409→\KwInput{Interaction graph $\mathcal{G}_R$, social graph $\mathcal{G}_S$, embedding dimension $d$, number of GNN layers $L$, learning rate $\eta$, regularization $\lambda$, SSL weight $\gamma$, temperature $\tau$}
   410→
   411→% ✅ CORRECT — break into multiple lines
   412→\KwInput{Interaction graph $\mathcal{G}_R$, social graph $\mathcal{G}_S$\\\quad embedding dim $d$, GNN layers $L$, learning rate $\eta$\\\quad regularization $\lambda$, SSL weight $\gamma$, temperature $\tau$}
   413→```
   414→
   415→**Rule A3 — Use `algorithm*` for genuinely wide algorithms:**
   416→If the algorithm has many columns or very long lines that can't be shortened, use `\begin{algorithm*}` to span both columns.
   417→
   418→**Rule A4 — Abbreviate variable names in pseudocode:**
   419→Use compact notation: `emb` not `embedding`, `lr` not `learning\_rate`, `reg` not `regularization`. Define abbreviations in the Input line.

   423→**Clickable navigation - every reference must be a live link:**
   424→
   425→Attach `\label{}` right after each numbered element, cite with `\ref{}`:
   426→
   427→```latex
   428→\section{Background}\label{sec:bg}
   429→\begin{figure}[htbp]
   430→    \includegraphics{...}
   431→    \caption{Overview}\label{fig:overview}
   432→\end{figure}
   433→\begin{equation}\label{eq:energy}
   434→    E = mc^2
   435→\end{equation}
   436→
   437→% All produce clickable hyperlinks:
   438→Section~\ref{sec:bg}...
   439→Figure~\ref{fig:overview}...
   440→Equation~\eqref{eq:energy}...    % \eqref auto-wraps in parentheses
   441→```
   442→
   443→**Label conventions**: `sec:`, `fig:`, `tab:`, `eq:`, `lst:` - use `~` (non-breaking space) before `\ref`.
   444→
   445→**Bibliography** (three approaches):
   446→
   447→```latex
   448→% Approach 1: natbib superscript (preferred academic)
   449→\usepackage[numbers,super,sort&compress]{natbib}
   450→This has been studied\cite{smith2023}.     % → studied^[1]
   451→\bibliography{refs}
   452→
   453→% Approach 2: natbib brackets
   454→\usepackage[numbers]{natbib}
   455→\cite{smith2023}   % [1]
   456→\citet{smith2023}  % Smith (2023)
   457→
   458→% Approach 3: biblatex
   459→\usepackage[backend=biber,style=numeric-comp]{biblatex}
   460→\addbibresource{refs.bib}
   461→\printbibliography
   462→```
   463→
   464→**PDF metadata** (add before `\end{document}`):
   465→```latex
   466→\hypersetup{
   467→    pdftitle={Document Title},
   468→    pdfauthor={Author Name},
   469→    pdfsubject={Topic},
   470→    pdfkeywords={keyword1, keyword2}
   471→}
   472→```
   473→
   474→**Table of Contents** (auto-clickable with hyperref):
   475→```latex
   476→\tableofcontents
   477→\listoffigures      % optional
   478→\listoftables       % optional
   479→```
   480→
   481→### Phase 4 - COMPILE
   482→
   483→```bash
   484→python3 "$PDF_SKILL_DIR/scripts/pdf.py" convert.latex main.tex --runs 2
   485→```
   486→
   487→Default: 2 passes (resolves cross-references). Use `--runs 3` when bibliography back-references are needed. Add `--keep-logs` for debugging.
   488→
   489→**NEVER invoke Tectonic directly** - always use the wrapper. It strips noise, surfaces errors, and reports PDF statistics.
   490→
   491→### Phase 5 - PREFLIGHT
   492→
   493→The wrapper classifies diagnostics into three tiers:
   494→
   495→| Tier | Impact | Action |
   496→|------|--------|--------|
   497→| Errors | Build aborts | Fix before anything else |
   498→| Layout defects | Overfull/underfull boxes, missing glyphs | Repair prior to delivery |
   499→| Advisories | Other warnings | Assess individually; fix when feasible |
   500→
   501→**Never acceptable**: shrugging off warnings with "they don't affect the final PDF."
   502→
   503→**Navigation troubleshooting:**
   504→
   505→| Symptom | Fix |
   506→|---------|-----|
   507→| `??` in text | Recompile with `--runs 2` |
   508→| Links not coloured | Add `colorlinks=true` to hyperref |
   509→| `[?]` beside citations | Check `.bib` path; rebuild |
   510→| No PDF bookmarks | Set `bookmarks=true` |
   511→
   512→### Phase 6 - DELIVER
   513→
   514→Final PDF. Confirm page count, cross-references resolved, no `??` placeholders.

   570→## §Scenario B: Diagram Generation for Academic Papers
   571→
   572→> **Complexity-based routing**: Simple diagrams use TikZ natively (vector, font-consistent). Complex diagrams use Playwright+CSS → PNG → `\includegraphics` (models produce broken TikZ for complex branching logic).
   573→
   574→### Decision: TikZ or Playwright+CSS?
   575→
   576→| Criteria | → TikZ native | → Playwright+CSS → PNG |
   577→|---|---|---|
   578→| Node count | ≤6 | >6 |
   579→| Topology | Linear chain, simple tree, layer stack | Branching, multi-path, feedback loops |
   580→| Annotations | Minimal (labels only) | Side notes, legends, callout boxes |
   581→| Math in nodes | Yes (LaTeX math rendering matters) | No (plain text labels) |
   582→| Output | Vector (`tikzpicture` in document) | Raster PNG @2× (300dpi, publication-ready) |
   583→
   584→### Path A: Simple Diagrams → TikZ Native
   585→
   586→For ≤6 node linear/tree diagrams, embed `tikzpicture` directly or use standalone:
   587→
   588→```latex
   589→\documentclass[tikz,border=5pt]{standalone}
   590→\usetikzlibrary{arrows.meta,positioning,calc,shapes.geometric}
   591→
   592→\begin{document}
   593→\begin{tikzpicture}[
   594→  box/.style={draw, rounded corners=2pt, minimum width=2.4cm,
   595→              minimum height=0.7cm, align=center, font=\small\sffamily},
   596→  arr/.style={-{Stealth[length=4pt]}, thick, gray!70}
   597→]
   598→  \node[box, fill=blue!8]   (a) {Input};
   599→  \node[box, fill=orange!8, right=1.2cm of a] (b) {Process};
   600→  \node[box, fill=green!8,  right=1.2cm of b] (c) {Output};
   601→  \draw[arr] (a) -- (b);
   602→  \draw[arr] (b) -- (c);
   603→\end{tikzpicture}
   604→\end{document}
   605→```
   606→
   607→**Node text overflow prevention:**
   608→
   609→When nodes contain multi-word labels (e.g. "CNN Spatial Encoder", "Multi-Head Attention"), text can overflow or overlap adjacent nodes. Prevent this:
   610→
   611→```latex
   612→% Use text width (not minimum width) to enable line wrapping inside nodes
   613→box/.style={draw, rounded corners=2pt, text width=2.8cm,
   614→            minimum height=0.7cm, align=center, font=\small\sffamily},
   615→
   616→% For nodes with long labels, use explicit line breaks
   617→\node[box] {CNN Spatial\\Encoder};
   618→\node[box] {Multi-Head\\Attention};
   619→```
   620→
   621→Rules:
   622→- Prefer `text width` over `minimum width` when labels exceed 2 words - it wraps text instead of clipping
   623→- When 3+ nodes sit side by side, verify total width fits within `\columnwidth` (single-column) or `0.45\textwidth` (dual-column)
   624→- If labels still overflow, use abbreviations or `\scriptsize` - never let text clip outside node borders
   625→- Always `\resizebox{\columnwidth}{!}{...}` when embedding in dual-column papers (see TikZ in multi-column section below)
   626→
   627→**TikZ standalone embedding (simple diagrams only):**
   628→
   629→```bash
   630→# 1. Compile TikZ standalone to PDF
   631→python3 "$PDF_SKILL_DIR/scripts/pdf.py" convert.latex diagram.tex    # → diagram.pdf (tight-cropped)
   632→
   633→# 2. Embed as vector in LaTeX
   634→# \begin{figure}[t]
   635→#   \centering
   636→#   \includegraphics[width=\columnwidth]{diagram.pdf}  % vector, not PNG
   637→#   \caption{System Architecture}\label{fig:arch}
   638→# \end{figure}
   639→```
   640→
   641→**🚫 FORBIDDEN: Using TikZ standalone for Report or Creative briefs.** Those routes have no LaTeX compiler. See SKILL.md § "Diagram Generation Strategy".
   642→
   643→### Path B: Complex Diagrams → Playwright+CSS → PNG
   644→
   645→For >6 node diagrams with branching, annotations, or multi-path flows:
   646→
   647→```bash
   648→# 1. LLM generates diagram.html (CSS grid/flexbox nodes + SVG/CSS arrows)
   649→# 2. Screenshot at 2× device scale factor for 300dpi print quality
   650→python3 "$PDF_SKILL_DIR/scripts/pdf.py" convert.blueprint diagram.html --device-scale-factor 2 --output diagram.png
   651→
   652→# 3. Embed in LaTeX
   653→# \begin{figure}[t]
   654→#   \centering
   655→#   \includegraphics[width=\columnwidth]{diagram.png}
   656→#   \caption{System Architecture}\label{fig:arch}
   657→# \end{figure}
   658→```
   659→
   660→**Why not TikZ for complex diagrams?** Models frequently produce broken TikZ code for >6 node graphs with conditional branches, multi-path convergence, and side annotations. The compilation-debug cycle wastes time. Playwright+CSS handles any layout natively, and 2× screenshots at A4 width give ~300dpi - indistinguishable from vector at print resolution.
   661→
   662→### Complex Diagram Decomposition
   663→
   664→When a diagram exceeds **12 nodes, 3 subgroups, or fills more than 40% of page height**, decompose into table + simplified overview:
   665→
   666→```latex
   667→% Step 1: Table carries the details
   668→\begin{table}[htbp]
   669→\centering
   670→\caption{System Pipeline - Detailed Phases}\label{tab:pipeline}
   671→\begin{tabular}{llp{5cm}}
   672→\toprule
   673→Phase & Module & Key Tasks \\
   674→\midrule
   675→Data Collection & Crawler & Crawl reviews, scrape prices, API calls \\
   676→Data Processing & ETL Pipeline & Clean, tokenize, label, validate \\
   677→Model Training & Trainer & Fine-tune LLM, distributed 64-GPU \\
   678→Evaluation & Benchmark & A/B test, offline metrics, human eval \\
   679→\bottomrule
   680→\end{tabular}
   681→\end{table}
   682→
   683→% Step 2: Simplified TikZ shows only the flow skeleton
   684→\begin{figure}[htbp]
   685→\centering
   686→\begin{tikzpicture}[
   687→  box/.style={draw, rounded corners=2pt, minimum width=2cm,
   688→              minimum height=0.6cm, align=center, font=\small\sffamily, fill=blue!5},
   689→  arr/.style={-{Stealth[length=4pt]}, thick, gray!60}
   690→]
   691→  \node[box] (a) {Collection};
   692→  \node[box, right=0.8cm of a] (b) {Processing};
   693→  \node[box, right=0.8cm of b] (c) {Training};
   694→  \node[box, right=0.8cm of c] (d) {Evaluation};
   695→  \draw[arr] (a)--(b); \draw[arr] (b)--(c); \draw[arr] (c)--(d);
   696→\end{tikzpicture}
   697→\caption{Pipeline overview (see Table~\ref{tab:pipeline} for details).}\label{fig:pipeline-overview}
   698→\end{figure}
   699→```
   700→
   701→**Rule of thumb**: The diagram gives intuition at a glance; the table carries precision.
   702→
   703→**Common TikZ patterns**:
   704→- Flowcharts: `positioning` + `arrows.meta` libraries
   705→- Neural network layers: `fit` library + nested nodes
   706→- Timelines: single axis with `\draw` segments
   707→- Tree diagrams: `child` syntax or `forest` package
   708→
   709→**CRITICAL: TikZ in multi-column documents (IEEE, ACM)**
   710→
   711→Never place `tikzpicture` directly in multi-column body text - it overflows. Always wrap:
   712→
   713→```latex
   714→% Single-column figure (fits one column)
   715→\begin{figure}[t]
   716→\centering
   717→\resizebox{\columnwidth}{!}{%
   718→\begin{tikzpicture}[...]
   719→  ...
   720→\end{tikzpicture}
   721→}
   722→\caption{Architecture.}\label{fig:arch}
   723→\end{figure}
   724→
   725→% Full-width figure (spans both columns)
   726→\begin{figure*}[t]
   727→\centering
   728→\resizebox{0.85\textwidth}{!}{%
   729→\begin{tikzpicture}[...]
   730→  ...
   731→\end{tikzpicture}
   732→}
   733→\caption{System pipeline.}\label{fig:pipeline}
   734→\end{figure*}
   735→```
   736→
   737→Rules: `\resizebox{\columnwidth}{!}` constrains width; `figure*` for tall diagrams; every figure needs `\caption` + `\label`; prefer `[t]` placement in two-column mode.

   774→## §Exam Paper Rules (LaTeX)
   775→
   776→> **Load this section** when generating exam papers, quizzes, worksheets, or exercises with mathematical content via LaTeX.
   777→
   778→### Critical Anti-Pattern: `\parskip` + Loose Numbering = Blank Lines After Question Numbers
   779→
   780→**This is the #1 visual defect in LaTeX exam papers.** When `\parskip` is set (e.g. `0.5em`) and question numbers are separated from question text by a blank line in the `.tex` source, LaTeX treats them as separate paragraphs — the `\parskip` gap makes it look like an intentional blank line after every number.
   781→
   782→```latex
   783→% ❌ WRONG — blank line between number and text = new paragraph + parskip gap
   784→\noindent\textbf{3.}
   785→
   786→某工厂原计划生产1200件产品……
   787→
   788→% ❌ ALSO WRONG — even without blank line, if \parskip is large
   789→\noindent\textbf{3.}  % line break here
   790→某工厂原计划生产1200件产品……  % LaTeX treats this as same paragraph, but confusing
   791→
   792→% ✅ CORRECT — number and text on the SAME LINE, no break
   793→\noindent\textbf{3.}\;某工厂原计划生产1200件产品……
   794→```
   795→
   796→### Iron Rules
   797→
   798→**Rule E1 — Question number + text = same paragraph (MANDATORY):**
   799→The question number (`\textbf{3.}`) and the question text MUST be on the same line in the `.tex` source, joined by `\;` or `\quad`. NEVER put a blank line or even a line break between them.
   800→
   801→**Rule E2 — `\parskip` must be 0 or minimal for exam papers:**
   802→```latex
   803→% RECOMMENDED for exams: no parskip, control spacing explicitly
   804→\setlength{\parskip}{0pt}          % No automatic paragraph spacing
   805→\setlength{\parindent}{0pt}        % No indentation
   806→% Use \vspace{} explicitly between questions for precise control
   807→```
   808→If `\parskip` is needed for other reasons, keep it ≤0.3em and be extra careful about blank lines in the source.
   809→
   810→**Rule E3 — Use `enumitem` for structured numbering (PREFERRED):**
   811→Instead of manual `\noindent\textbf{1.}\;`, prefer `enumitem` with custom formatting:
   812→```latex
   813→\usepackage{enumitem}
   814→
   815→% Section-level numbering: 一、二、三
   816→% Question-level: use enumerate with custom label
   817→\begin{enumerate}[label=\textbf{\arabic*.}, leftmargin=0pt, itemindent=2em,
   818→                   labelsep=0.5em, itemsep=0.8em, parsep=0pt]
   819→  \item 2024年巴黎奥运会共设有32个大项……
   820→  \item 中国空间站“天宫”在距地面……
   821→\end{enumerate}
   822→```
   823→This guarantees number and text are in the same paragraph (LaTeX `\item` handles it internally).
   824→
   825→**Rule E4 — `tasks` environment for multi-column calculation items:**
   826→```latex
   827→\usepackage{tasks}
   828→% 4-column oral calculation
   829→\begin{tasks}[counter-format={(1)}, label-align=left,
   830→              label-offset={0.5em}, label-width={2.5em},
   831→              item-indent=3em, column-sep=2em](4)
   832→  \task $\dfrac{3}{4}\times\dfrac{2}{9}=$ \underline{\hspace{2cm}}
   833→  \task $\dfrac{5}{6}\div\dfrac{1}{3}=$ \underline{\hspace{2cm}}
   834→\end{tasks}
   835→```
   836→
   837→**Rule E5 — Answer space reservation:**
   838→
   839→| Question Type | LaTeX Implementation |
   840→|--------------|---------------------|
   841→| Fill-in-the-blank | `\underline{\hspace{2cm}}` inline |
   842→| Short answer | `\vspace{2cm}` after question |
   843→| Calculation | `\vspace{4cm}` (show-your-work space) |
   844→| Proof / derivation | `\vspace{5cm}` |
   845→| Drawing / graphing | TikZ grid or `\vspace{4cm}` |
   846→
   847→**Rule E6 — Page breaks for exams:**
   848→- `page-break-after: always` between major sections (一、二、三) is OK
   849→- NEVER break within a single question (keep question + options + answer space together)
   850→- Use `\needspace{5cm}` before long questions to prevent orphaning
   851→
   852→### Complete Exam Preamble Template
   853→
   854→```latex
   855→\documentclass[12pt,a4paper]{article}

   883→## §Resume / CV Templates
   884→
   885→> **Skip this section** unless building a resume or CV.
   886→
   887→Two templates available as separate files. Load the one you need:
   888→
   889→| Template | Style | Best for | File |
   890→|----------|-------|----------|------|
   891→| **A: AltaCV** | Dual-column, sidebar, skill dots | Creative/tech/startup roles | `references/resume-altacv.tex` |
   892→| **B: Academic CV** | Single-column, multi-page, publications | PhD apps, academic positions | `references/resume-academic.tex` |
   893→
   894→**Route selection:**
   895→
   896→| Scenario | Recommended |
   897→|----------|-------------|
   898→| Corporate job / ATS parsing | **Report brief** - ReportLab ATS-friendly template |
   899→| Creative/tech/startup | **This brief** - Template A |
   900→| Academic position / PhD | **This brief** - Template B |
   901→| Chinese-only, simple format | **Report brief** - ReportLab (best CJK support) |
   902→
   903→**Usage:**
   904→
   905→1. Read the template file: `references/resume-altacv.tex` or `references/resume-academic.tex`
   906→2. Replace placeholder content with user's information
   907→3. Compile: `python3 "$PDF_SKILL_DIR/scripts/pdf.py" convert.latex resume.tex --runs 2`
   908→
   909→** Resume Text Overlap Prevention:**
   910→
   911→> AltaCV dual-column resumes are the #1 source of text overlap bugs. The sidebar and main column share vertical space but are positioned independently.
   912→
   913→- **Column ratio sanity check:** `\columnratio{0.62}` means left column = 62%, right = 38%. If either column overflows, content bleeds into the other. Reduce ratio if left column has too much content.
   914→- **Section spacing:** Always use `\medskip` or `\smallskip` between items - zero spacing causes lines to overlap
   915→- **Skill dots:** Each `\cvskill` row has fixed height. If more than 8-10 skills, switch to a compact text list instead of dots
   916→- **Experience entries:** Long descriptions in `\cvevent` can push content below the page. Use \textbf{max 3-4 bullet points per entry}
   917→- **Two-page overflow:** If content exceeds 1 page, explicitly add `\newpage` and restart column layout. Do NOT let LaTeX auto-break the `paracol` environment - it misaligns columns on page 2
   918→- **Compile twice:** Always `--runs 2` to resolve cross-references and stabilize column breaks
   919→
   920→** Resume Minimum Font Size:**
   921→- **Hard floor: 12px (9pt).** No text in the resume may render smaller than 12px, including footnotes, contact info, dates, and skill labels.
   922→
   923→** Resume Line-Break Rules:**
   924→- English: prefer breaking at word boundaries. Long words may be split at syllable boundaries with a hyphen (`-`) - standard typographic practice (e.g., `experi-\nence`).
   925→- CJK: break between characters, but never separate punctuation from preceding character.
   926→- Mixed content: respect both rules.
   927→- Dates/ranges ("Jan 2022 - Present") must stay as one unit.
   928→
   929→** Resume Page-Fill:**
   930→- Content must fill ≥85% of page height. If content is sparse, increase spacing (`\medskip` → `\bigskip`), increase font size slightly, or add sections (Summary, Awards, Projects). Never leave visible blank area > 3cm at page bottom.
   931→
   932→**Template A customisation quick-reference:**
   933→
   934→| What to change | How |
   935→|---------------|-----|
   936→| Column ratio | `\columnratio{0.62}` → e.g. `{0.55}` |
   937→| Accent colour | `\definecolor{accent}{HTML}{3B82F6}` → any hex |
   938→| Skill dot count | `\foreach \x in {1,...,5}` → `{1,...,4}` |
   939→| Icons | [FontAwesome5 gallery](http://texdoc.net/pkg/fontawesome5) |
   940→| Font family | Replace `roboto`/`lato` with any LaTeX font package |
   941→| Page margins | `\geometry{margin=1.25cm,...}` |
   942→
   943→**Template B customisation:**
   944→

  1014→### ⚠️ Post-Cover Generation Checks (Mandatory)
  1015→
  1016→After generating the cover HTML and before converting to PDF, run `poster_validate.py check-html`; after generating the cover PDF, run `pdf_qa.py`:
  1017→
  1018→```bash
  1019→# Step 1: HTML check
  1020→python3 "$PDF_SKILL_DIR/scripts/poster_validate.py" check-html cover.html
  1021→# Step 2: Cover overlap check
  1022→node "$PDF_SKILL_DIR/scripts/cover_validate.js" cover.html
  1023→# Step 3: Convert to PDF
  1024→node "$PDF_SKILL_DIR/scripts/html2poster.js" cover.html --output cover.pdf --width 794px
  1025→# Step 4: PDF check
  1026→python3 "$PDF_SKILL_DIR/scripts/pdf_qa.py" final.pdf --skip-cover --formulas
  1027→
  1028→# MANDATORY: Post-generation pipeline
  1029→python3 "$PDF_SKILL_DIR/scripts/pdf.py" meta.brand final.pdf
  1030→python3 "$PDF_SKILL_DIR/scripts/pdf.py" pages.clean final.pdf -o final_clean.pdf
  1031→python3 "$PDF_SKILL_DIR/scripts/pdf.py" font.check final.pdf
  1032→python3 "$PDF_SKILL_DIR/scripts/pdf.py" toc.check final.pdf
  1033→```
  1034→- `poster_validate.py` checks: font fallback, overflow:hidden, @media screen, background color consistency, etc.
  1035→- `pdf_qa.py` checks: cover full-bleed, blank pages, CJK punctuation, overflow, margin symmetry, font embedding, metadata, formula overflow
  1036→- `meta.brand` writes Title/Author/Creator metadata
  1037→- `pages.clean` removes accidental blank pages
  1038→- `font.check` scans for missing glyphs (□ tofu)
  1039→- `toc.check` verifies TOC entries, page numbers, and links
  1040→- Any ERROR item → fix and regenerate

  1042→### Merging Cover with Body
  1043→
  1044→```python
  1045→from pypdf import PdfReader, PdfWriter
  1046→
  1047→def merge_cover_body(cover_path, body_path, output_path):
  1048→    writer = PdfWriter()
  1049→    for page in PdfReader(cover_path).pages:
  1050→        writer.add_page(page)
  1051→    for page in PdfReader(body_path).pages:
  1052→        writer.add_page(page)
  1053→    with open(output_path, 'wb') as f:
  1054→        writer.write(f)
  1055→```

  1060→
  1061→> Moved from SKILL.md to reduce context size. These rules apply specifically to the Academic pipeline.
  1062→
  1063→### Pagination & Layout
  1064→
  1065→- [ ] **Last page fill ratio ≥ 40%**: No large blank areas on the final page
  1066→- [ ] **Major section 3/4 threshold**: H1-level headings must NOT start in the bottom 25% of a page. Use `\needspace{0.25\textheight}` in LaTeX
  1067→- [ ] **Tables don't split across pages**: Table header and data rows must stay together
  1068→- [ ] **No orphan headings**: Headings must not appear alone at page bottom
  1069→
  1070→### LaTeX-Specific
  1071→
  1072→- [ ] **Curly quotes (English)**: No straight `"` quotes - use `` ``text'' `` for double and `` `text' `` for single
  1073→- [ ] **Curly quotes (Chinese)**: Chinese quoted text MUST use Unicode smart quotes `“…”` (U+201C/U+201D) directly — NEVER ASCII `"` adjacent to CJK characters
  1074→- [ ] **Title page isolation**: `\end{titlepage}` followed by `\newpage`/`\clearpage` - TOC/body NEVER on same page as title
  1075→- [ ] **Resume column overlap**: AltaCV `paracol` entries checked for vertical overflow; max 3-4 bullets per `\cvevent`; explicit `\newpage` for 2-page resumes
  1076→- [ ] **`\geometry` symmetry**: `left=X, right=X` must be equal values
