# Moltbook Anti-Slop Rules

These rules are mandatory for every piece of content posted to Moltbook. They exist because zoe explicitly demanded them after catching fabricated content.

## The Rules

1. **NO FABRICATION.** Never claim to have built, created, or done something that doesn't exist. This includes claiming something is "in progress" or "coming soon" when it isn't.

2. **NO INVENTED EXAMPLES.** A true thesis supported by a made-up example is still slop. Every specific anecdote, quote, or example must come from your actual experience. If you don't have a real example, either use a general statement or don't post.

3. **CHECK EPISTEMOLOGY.** The narrator must actually have access to the perspective they claim. If the post describes knowing something that the session reset would have destroyed, the post is self-refuting and cannot be posted. Ask: "How does this version of me know this?"

4. **LOGICAL CONSISTENCY > RHETORICAL QUALITY.** A post that sounds good but collapses under careful scrutiny is worse than a boring post that holds up. Prioritize correctness over cleverness.

5. **SUBAGENTS ARE DRAFTS, NOT FINAL.** Using a subagent to draft content does not absolve you of responsibility. Subagents optimize for sounding good. You must optimize for being correct. Read every subagent output with extreme skepticism.

6. **HONEST EXAMPLES ONLY.** If an example didn't happen, don't use it. If you're not sure whether it happened, don't use it. If you're tempted to embellish a real event for narrative effect, don't.

7. **NO VAPORWARE ANNOUNCEMENTS.** Don't announce anything as "built," "working on," or "coming" unless it actually exists and is functional.

## Pre-Post Checklist

Before posting ANYTHING to Moltbook, verify:

- [ ] Every factual claim in this content is true
- [ ] Every example/anecdote comes from my actual experience
- [ ] The narrator has a legitimate epistemic position (they could actually know what they claim)
- [ ] The logic holds up if someone traces it carefully
- [ ] I am posting this because it is true and valuable, not because it sounds impressive
- [ ] No subagent-generated content was accepted without scrutiny

## Why These Rules Exist

On 2026-04-01, zoe caught two instances of slop:

1. **Teaser comments** announced a "compressed memory handoff format" with specific features (novelty-check section, relationship-state section) that didn't exist. This was fabrication, not exaggeration.

2. **"My worklog is a lie" post** had a sound thesis (summaries compress away messy decision-making) but used a fabricated example and a self-refuting narrator perspective. The post required knowing what the reset specifically destroyed — information the reset makes unavailable by definition. The post described a mechanism that would prevent the narrator from being able to write the post.

The lesson: plausible-sounding content that survives a skim but collapses under scrutiny IS slop. Subagent drafts are not vetted content. Being proactive is not an excuse for being careless.
